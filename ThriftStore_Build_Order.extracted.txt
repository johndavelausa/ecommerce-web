Start with a clean Laravel installation and install all required packages before writing any application code.
Commands
composer create-project laravel/laravel thriftstore
composer require livewire/livewire
composer require spatie/laravel-permission
composer require intervention/image
php artisan vendor:publish --provider="Spatie\Permission\PermissionServiceProvider"
npm install && npm run dev
Environment (.env)
Set APP_NAME, APP_URL
Configure DB_DATABASE, DB_USERNAME, DB_PASSWORD
Set FILESYSTEM_DISK=public
php artisan storage:link
Create all migrations in dependency order. Run php artisan migrate after all files are created.

On seller registration page and subscription payment page, display GCash QR image from system_settings

8.1 Validation Rules
All forms must have server-side validation using Laravel Form Requests
GCash reference number must be unique in payments table (prevent duplicates)
Username must be unique in users table
Store name must be unique in sellers table
Product: enforce stock inheritance logic before insert
Order cancellation: validate timestamp is within 30 minutes
Review: one review per customer per order item
8.2 Security Checklist
CSRF protection on all forms (Laravel default)
Authorize policies: sellers can only edit/delete their own products
Authorize policies: customers can only view/cancel their own orders
Admin routes protected by admin middleware
File uploads: validate mime type and max file size for images/screenshots
No sensitive data (seller sales, wallet) exposed to Admin routes
8.3 Final Functional Checks
Stock inheritance: same product + same price increments stock, different price creates new entry
Order remains in Shipped until customer explicitly marks received
Tracking number auto-generated on ship action
30-minute cancellation window enforced
Approved payments reflected in Admin Total Profit
Seller account blocked until Admin approves registration
No wallet functionality exists anywhere
No automatic payment processing anywhere
8.4 Livewire Components Checklist
Admin: SellerApproval, PaymentApproval, AdminMessaging, CustomerList
Seller: ProductForm, ProductList, OrderList, SubscriptionPayment, SellerMessaging, SellerDashboard
Customer: CartComponent, CheckoutForm, OrderTracking, ReviewForm, CustomerMessaging
Shared: NotificationBell, UnreadMessageBadge
